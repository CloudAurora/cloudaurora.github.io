HTTPS

服务器证书的验证

操作系统本地会存储信任机构的根证书，服务器使用这些机构颁发的证书办法给用户时，才会被用户操作系统认证为合法证书

HTTPS连接过程

  1. 用户发送给服务器一个随机码
  2. 服务器返回用户随机码以及服务器的公钥
  3. 用户通过非对称加密方式加密随机码后发送给服务器
  4. 连接建立后通过对称加密方式通信，对称加密的密钥通过随机码生成

非法的证书颁发机构

一些机构会颁发一些假证书，如果这些机构的根证书被安装在本地，则这些假证书会被操作系统认为合法，如果中间人将假证书发送给用户，操作系统会认为证书合法，从而造成用户的所有数据都使用中间人的公钥加密，从而使得中间人可以解密所有数据

ARP与ARP攻击

  • ARP是根据IP地址获取MAC地址的一个TCP/IP协议

  • ARP攻击的原理是对某IP的解析返回欺骗者的MAC地址，从而造成IP对应不到正确的设备上

渗透工具

  • 基于Debian的Kali Linux内置了一系列渗透软件

  • 用于渗透的PC软件

    1. Ettercap 进行ARP和中间人攻击
    2. BeEF 进行XSS攻击
    3. Metasploit 安全漏洞检测工具
    4. Aircrack-ng 用来破解WiFi，WEP加密容易被破解，目前比较安全的是WPA2-PSK，WiFi破解原理是截获握手包进行离线破解，即使使用低压CPU都可以进行每秒5000次以上的破解，破解关键看密码库的强度
    5. Reaver 通过PIN码方式破解WiFi，原理是路由器支持一种8位PIN码认证，PIN码只有10万种左右的情况，为了安全最好关闭PIN功能
    6. SQLMap 数据库注入检测工具

  • 用于渗透的移动端软件

    1. dSploit 主要用来进行嗅探和中间人攻击
    2. zANTI 比dSploit更美观，有更多功能
